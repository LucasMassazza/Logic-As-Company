Relevamiento

Cursograma:

El cliente realiza una compra, se genera un archivo denominado Solicitud de Compra (SC). Luego, se consulta a una Base de Datos (BDD), si el cliente que realizo la consulta es moroso o no. Si lo es, le informamos que es moroso y se elimina la solicitud de la compra. Si no lo es, se genera una copia de SC, llamada SC1, y la original se guarda como registro de la solicitud de compra. Con la SC1 la enviamos a Finanzas, en la cual se ve la forma de financiacion que va a tener esta SC1. Luego de elegir el tipo de finaciacion, se genera una factura, y una copia. La Factura (Fac) original se envia a una BDD para tener un registro de la compra. La copia 1 se manda al almacen, para poder retirar el producto. Aca se consulta la disponibilidad. Si no la hay, se genera un pedido al proveedor (Ped). Este proveedor recibe el pedido, envia el producto y a su vez un archivo denominado Producto Recibido (PR). Si llega a haber disponibilidad, se generan dos archivos. Uno es el de RP, el cual se almacena en una base de datos, para su control. Y el otro es el de Generar Pedido (GP), el cual es un archivo que va en conjunto a la factura con el producto. Estos dos archivos, se envian a ventas para su control. El archivo GP se guarda en una BDD. Y la Fac1, se la envia al cliente , junto al producto.
